Основные команды при работе с Git

Команды настройки программы

git config --global == глобальные настройки
git config --global == глобальные настройки
git config --global user.name == создание имени пользователя
git config --global user.email == задание e-mail адреса пользователя
git config --list == просмотр текущих настроек

Работа с Git

git init == инициализация нового репозитория в текущей папке
git status == получить информацию о текущем состоянии репозитория
git reflog == очень полезная команда, позволяет переместиться по истории изменений (почти в любом случае). кроме того наглядно показывает относительно чего были сделаны изменения - переход по ветке, переименование ветки, создание коммита и пр. Чтобы изменить нужное состояние, нужно запустить checkout, используя абсолютную (git checkout 1234568) или относительную (git checkout HEAD@{3}) ссылку
git log == подробный лог коммитов
git log --oneline == сокращенный лог коммиитов
git log -n 3 --oneline == сокращенный лог последних трех коммитов (количество коммитов указывается вручную)
git log --oneline --decorate == --oneline сокращает количество занятых строк, а опция --decorate добавляет относительные имена (локальные/удалённые ветви + HEAD)
git log --graph == выведет график, показывающий структуру ветвления истории коммитов
git log -p == подробный лог изменений КАЖДОГО файла
git diff == просмотр списка изменений, внесенных в репозиторий
git diff --staged == просмотр подготовленных изменений
git diff "name.*" == подробный просмотр изменений в файле name.*
git add == добавление файла в индекс отслеживания
git add . == добавляет все файлы в индекс
git commit == создание коммита (обязательно д.б. сообщение)
git commit -ad "comment" == ДВА В ОДНОМ: добавляет файл в индекс и оставляет комментарий
git commit --amend -m "new_massage" == перезаписывает комментарий последнего коммита (если допущена ошибка в сообщении, снимок коммита не меняется)
GIT RESET == | С ЭТИМИ КОМАНДАМИ НАДО РАЗОБРАТЬСЯ
git revert == отменяет действия прошлых коммитов, создавая новый, содержащий все отмененные изменения. (безопасный способ, используется в публичных ветвях, где изменение истории нежелательно)
git remote -v == посмотреть статус соединения репозиториев
git remote add == {{{{{(((дописать)))}}}}}
git push origin master == отправить изменения в удаленный репозиторий
git pull == подтягивает изменения в удаленном репозитории в локальный, на комп
git pull --verbose == просмотреть подробные сведения о загруженных файлах

Работа с ветками

git branch == посмотреть список веток
git branch "name" == создание ветки name
git branch -d "name" == удаление ветки name
git branch -D "name" == принудительное удаление ветки name, даже с не слитыми изменениями
git push origin --delete "name" == удаление из удаленного репозитория ветки name
git branch -a (-r) == показать список всех удаленных веток
git branch -m new_name == переименовать текущую ветку
git checkout == команда для переключения между ветками, коммитами
git checkout name_branch == переход к ветке name_branch
git checkout -b name_branch == СОЗДАТЬ и сразу ПЕРЕЙТИ на ветку name_branch
git merge "name" == команда выполнит слияние текущей ветки с веткой name (произойдет вливание ветки name)
git merge --no-ff "name" == команда объединит заданную ветку с основной и произведёт коммит слияния. Это необходимо для фиксации всех слияний в репозитории.
git merge --abort == отмена слияния веток
git branch --merged == список всех веток, соединенных с текущей
ls == просмотр списка файлов
clear == очистить экран
cd == переход по директориям
cd .. == переход на уровень "выше" (пробел обязателен!)
cd ../.. == переход на два уровня "выше"
cd "name" == переход в папку с именем "name". Прим - должна находиться в текущей папке, в ином случае - указывать полный путь!
cd ~ == переход в стартовую папку
mkdir "name" == создание директории
touch "name.*" == создание файла (* - расширение файла)
cp name_1.* name_2.* == копировать файл name_1.* в новый файл name_2.*
mv name_1.* name_2.* == переименовать name_1.* в name_2.*
? mv также позволяет перемещать файлы, пока не понял как
rm == удаление файлов и дирректорий
rm " .*" == удаление всех файлов с расширением .*
echo "text" > name.* == вносит текст text в файл name.*
cat name.* == выводит содержимое файла name.* в консоль
